# CLAUDE.md — контекст проекта

## Что это за проект

Сценарный RAG-бот для стоматологической клиники. Концепция: не просто чат с нейросетью, а **управляемый воронкой продаж бот** — «эмпатичный продажник», который сам подсказывает пользователю следующий шаг.

## Текущая версия: 1.0 (demo-prod, один клиент)

Задача — запустить в прод для демонстрации клиентам. Контент базы неполный, заказчик не предоставил всё — работаем с тем что есть.

## Архитектура кратко

- **Backend:** Python/Flask, SQLite сессии, 1 воркер (gunicorn)
- **RAG:** dual retrieval (embeddings + rewrite), alias channels (raw/lemma/trigram), selective LLM rerank
- **Ценовой слой:** `service_catalog.json` + `prices.json` на клиента, детерминированные цены (LLM не трогает цифры)
- **Intent routing:** regex → LLM классификатор (`price_lookup` / `price_concern` / `other`)
- **Widget:** JS виджет в `static/widget/`, подключается на сайт клиента
- **Клиент:** `clients/default/`

## Ключевые принципы (важно соблюдать)

- LLM приоритетнее regex если даёт лучший результат — лишние токены не проблема
- Цены никогда не генерирует LLM — только из `prices.json` или `*__pricing__*.md`
- Не усложнять логику ради редких edge cases — лучше перевести на администратора
- Не упираться в экономию токенов в важных диалоговых сценариях

## Что намеренно не делаем в v1.0

- CRM / n8n / webhook интеграции
- Мультиклиентная изоляция
- Стриминг на бэкенде
- Большой рефакторинг app.py
