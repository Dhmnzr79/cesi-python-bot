## Справочник: все заготовки текста бота (код)

Ниже — **фиксированные строки**, которые попадают пользователю в **`answer`** (или в **текст CTA** из дефолта), плюс **тестер**. Ответы из **LLM по контенту** сюда не входят. Подписи **meta** в JSON — машинные ключи, не показываются как реплика (если фронт сам не выводит).

### `app.py` — словарь `TXT` (передаётся в `flow_handlers.handle_flows`)

| Ключ | Когда показывается | Текст |
|------|-------------------|--------|
| `lead_name_prompt` | Явный **интент записи** (`booking_intent`), или клик **CTA с действием lead** (`cta_action == "lead"`), или ответ **«да»** после предложения CTA (`last_bot_action == "offered_cta"`). Старт сбора имени. | Отлично. Как к вам можно обращаться? |
| `lead_name_retry` | Lead-flow, шаг `collecting_name`: ввод **не распознан как имя** (`extract_name` → `None`). | Как к вам можно обращаться? Напишите, пожалуйста, имя. |
| `lead_phone_prompt_tpl` | После успешного имени; `{name}` подставляется. | Спасибо, {name}. Оставьте номер телефона для связи. |
| `lead_phone_retry` | Lead-flow, шаг `collecting_phone`: номер **не распознан** (`extract_phone` → `None`). | Не получилось распознать номер. Напишите телефон в формате +7XXXXXXXXXX. |
| `lead_submit_ok` | После успешной записи лида на диск (`handle_lead` → 200). | Принято, передали заявку администратору. С вами свяжутся. |
| `lead_submit_error` | `handle_lead` вернул не 200 (ошибка сохранения / валидации на стороне сервиса лида). | Не удалось сохранить заявку. Проверьте номер телефона и попробуйте ещё раз. |
| `situation_prompt` | Кнопка/действие **«ситуация»** (`situation_action == start` или `action == situation`), или **«да»** после предложения блока ситуации (`last_bot_action == offered_situation`). | Понимаю, что у каждого ситуация своя. Напишите коротко, что вас беспокоит или какой у вас вопрос, буквально в 1–2 фразах. |
| `situation_retry_short` | В режиме ожидания ситуации (`situation_pending`): пустой или слишком короткий ввод. | Чтобы помочь точнее, напишите коротко вашу ситуацию в 1–2 фразах. |
| `situation_to_lead_name` | После того как пользователь отправил текст ситуации (не пустой, длина ≥ 3): переход к сбору имени. | Спасибо. Как к вам можно обращаться? |
| `situation_back_fallback` | Действие **«назад»** из ситуации (`situation_action == back`), но **нет** сохранённого снимка последнего контентного UI — показываем общий выход. | Хорошо, продолжим. Задайте вопрос или выберите тему. |

### `app.py` — прочее (не `TXT`, но влияет на LLM)

| Когда | Текст |
|-------|--------|
| Запрос по **`ref`** без текста вопроса: подстановка в LLM как «вопрос» | Информация из {ref} |

### `ux_builder.py` — ответы без retrieval / ошибки

| Функция | Когда | Текст в `answer` |
|---------|--------|------------------|
| `empty_question_response` | Пустой `q` в `/ask` (после flow), до поиска. | Уточните вопрос. |
| `no_candidates_response` | Нет кандидатов retrieval / невалидный chunk после selection. | Пока не нашёл подходящий материал в базе. Сформулируйте вопрос иначе или выберите один из вариантов ниже. |
| `reset_session_response` | Команды сброса сессии в теле запроса: `/reset`, `/новая`. | Начнём заново. Чем помочь? |
| `internal_error_response` | Исключение в обработчике `/ask` (`ask_failed` в логах). | Извините, не получилось ответить. Попробуйте переформулировать вопрос. |
| `low_score_response` | Режим `low_score` в `query_selector` (низкий score top-1). | Не нашёл точного ответа на этот вопрос. Уточните или выберите тему. |

**CTA при `low_score`:** не строка в `ux_builder`, а **`config.default_cta_dict()`** — текст из переменной окружения **`DEFAULT_CTA_TEXT`** (по умолчанию в коде: «Записаться на консультацию»), действие **`DEFAULT_CTA_ACTION`** (по умолчанию `lead`).

**Успешный ответ с чанком:** поле `answer` генерирует **LLM**; заготовки выше не используются. Текст кнопки **CTA** при обычном ответе берётся из **метаданных документа** (`meta.cta_text` / `cta_action` в md/yaml), иначе CTA может быть `null`.

### `chunk_responder.py` — запасной ответ

| Когда | Текст |
|-------|--------|
| LLM вернул пустой/пробельный `answer`, при этом в чанке есть текст — подставляется обрезка контента; если и её нет | Пока не нашёл точный ответ. Можете уточнить вопрос? |

### `static/tester.html` — только UI тестера (не ответ API)

Заголовки, плейсхолдеры, подписи кнопок отладки: «Тестер бота», «Введите вопрос», «Отправить», «Новая сессия», блоки «Диалог», «Кнопки от бота», «Видео», «Ситуация», «Сырой JSON ответа», «Кандидаты retrieval»; «Показать видео» (fallback подписи видео); «Пока заглушка: здесь позже покажем видео-блок…»; «Сейчас блок ситуации не предложен ботом.»; «Режим ситуации: опишите коротко ваш случай…»; placeholder textarea ситуации; «Отправить ситуацию», «Назад к диалогу», «Рассказать о ситуации»; «Загрузка…», «Загрузка candidates…»; «Не удалось загрузить retrieval candidates»; «Ошибка при запросе к /ask»; «Ошибка загрузки candidates»; при пустом `answer` в ответе: «(пустой ответ)».

### Не пользовательские реплики (для полноты картины)

- **`llm.py`**: системные и user-промпты к модели (роль врача, правило JSON, «Вопрос пациента:», «Контент для ответа…») — **не** поле `answer` в API.
- **`session.mem_context`**: префикс «Недавний диалог:» внутри промпта — пользователю не показывается.
- **`/lead`**, ошибки `jsonify` в роутах (`unknown_client`, `bad_json` и т.д.) — машинные/служебные, не чат-бот.

*Актуальность: сверено с репозиторием по файлам `app.py`, `flow_handlers.py`, `ux_builder.py`, `chunk_responder.py`, `config.py`, `static/tester.html`.*
